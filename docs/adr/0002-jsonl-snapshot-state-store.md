# ADR 0002: 第一版状态存储采用 JSONL 事件日志加快照

## 背景

Alice 需要一个可审计、可恢复、实现成本可控的状态存储方案，覆盖：

- 任务状态
- 外部事件 dedupe
- `outbox`
- 审核请求与确认对象
- 启动恢复

## 决策

第一版采用：

- JSONL append-only 事件日志
- 周期性快照
- 投影和附属对象单独物化

其中：

- `outbox` 队列
- 外部事件 dedupe 索引

都必须由事件日志重放派生，不允许再引入第二个独立耐久化提交边界

恢复方式：

- 先加载最新快照
- 再重放后续事件
- 最后对账 `outbox`、审核和评测对象

## 备选方案

### 方案 A：直接使用 SQLite

优点：

- 查询能力强

缺点：

- 第一版会较早绑定表结构
- 事件溯源语义不够直观

### 方案 B：完全内存态加周期导出

优点：

- 实现简单

缺点：

- 恢复能力差
- 无法满足审计与对账需求

## 影响

- 事件是最终审计事实
- 投影损坏可以通过重放恢复
- 写路径必须先成功 append，再允许推进状态
- `outbox` 与 dedupe 索引即使物化文件丢失，也必须能由 replay 重建

## 后续事项

- 明确日志切段、压缩和 compact 规则
- 后续若切换 SQLite / BoltDB，需保持事件语义不变
