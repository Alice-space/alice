# ADR 0003: 所有外部副作用采用 outbox-first 与协议级幂等

## 背景

Alice 会频繁与 GitHub、GitLab、Cluster、控制面交互。外部系统都可能：

- 超时
- 返回成功但本地回写失败
- 重复投递回执
- 在恢复后再次收到同一动作

如果没有统一的副作用执行模型，系统会出现重复建 issue、重复 merge、重复发作业等问题。

## 决策

采用：

- 所有外部副作用先在事件日志中写入 `OutboxActionCreated`
- 通过 `task_id:event_id:action_type` 生成幂等键
- 幂等键作为 MCP 协议字段强制透传到底层
- 通过恢复对账修复“外部成功、本地未回写”的情况

补充约束：

- `outbox` 是由事件日志派生出的执行队列，不是独立真源
- 只读 MCP 查询不进入 `outbox`，仅有副作用或必须对账的动作进入 `outbox`

## 备选方案

### 方案 A：业务代码直接调用外部 API

优点：

- 实现快

缺点：

- 无法统一恢复
- 重复执行风险高

### 方案 B：依赖外部平台自身防重

优点：

- 本地逻辑少

缺点：

- 不同平台能力不一致
- 无法统一审计

## 影响

- 所有外部动作都会晚一步执行，但换来可恢复性
- MCP 协议必须承载幂等元数据
- 恢复流程需要实现外部状态查询能力

## 后续事项

- 固定各 `ActionType` 的 payload schema
- 为每个 MCP 域补错误码与重试语义
