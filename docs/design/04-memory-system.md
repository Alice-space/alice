# 04. 记忆系统设计

## 1. 目标

记忆系统用于提升长期系统效率和一致性，解决两个问题：

- 避免每次任务都从零开始。
- 避免历史失败和用户偏好只存在于日志或对话里，无法复用。

记忆系统不是：

- 原始日志数据库。
- 完整文档存储替代品。
- 不加筛选的“所有内容都记住”系统。

## 2. 三层记忆模型

### 2.1 运行工作记忆

- 作用域：单次 `run`。
- 生命周期：短。
- 内容：当前思路、局部发现、恢复摘要、待回写候选。
- 落地方式：可持久化为 `MemoryRecord(scope=run)`，但默认不视作长期知识。

### 2.2 项目记忆

- 作用域：单个项目。
- 生命周期：中长期。
- 内容：项目目标、实验基线、失败模式、已证伪路线、仓库约定、环境约束。
- 用途：减少同项目反复试错。

### 2.3 全局记忆

- 作用域：整个系统。
- 生命周期：长期。
- 内容：用户偏好、通用 playbook、故障处理经验、默认通知风格、常见审批规则。
- 用途：跨项目复用。

## 3. 记忆写入策略

### 3.1 可写入来源

- 用户明确给出的偏好和规则。
- 人工审批、驳回、改方向的决定。
- 评测闭环中稳定出现的基线和失败模式。
- 长期探索中明确证伪或证实的路线。
- 恢复流程产出的稳定恢复摘要。

### 3.2 不应直接写入长期记忆的内容

- 单次模型猜测。
- 低置信度的假设。
- 未验证的“可能有用”结论。
- 大段原始日志。
- 仅在本轮运行临时有效的 shell 输出。

### 3.3 晋升规则

- `run -> project`：需要满足“可复用 + 已验证 + 明确影响成功率”。
- `project -> global`：需要满足“跨项目复用 + 足够稳定 + 不是项目私有知识”。
- `evaluation_baseline` 类型只有在评测产物可比较时才允许晋升。

## 4. 检索流程

### 4.1 Recall 输入

- 当前 `task_type`
- `project_id`
- 关键词
- 作用域偏好
- token 预算
- 当前角色

### 4.2 Recall 输出

- 最相关的少量记忆摘要。
- 每条记忆的来源、置信度、新鲜度。
- 冲突记忆标志。

### 4.3 检索约束

- 不允许把整个记忆库完整注入上下文。
- 回忆结果必须受 token 预算限制。
- 冲突记忆不能直接混入“真相”，必须标出冲突。
- 记忆命中不能绕过项目策略、审批或正式产物。

## 5. 存储形态

首版逻辑上分两部分：

- 元数据存储
  保存 `MemoryRecord` 主字段、标签、时间戳、置信度。
- 内容存储
  保存长文本或引用指针，例如 Markdown 报告、PR 评论、实验摘要。

可选增强：

- 向量索引。
- 标签倒排索引。
- 基于项目和类型的二级缓存。

## 6. Freshness 与衰减

### 6.1 新鲜度

记忆需要具备新鲜度，不同类型规则不同：

- 用户偏好：通常稳定，但可被新输入覆盖。
- 环境约束：容易过时，需要周期验证。
- 评测基线：代码、数据或环境变更后可能失效。

### 6.2 衰减

当记忆满足以下条件时应降权或归档：

- 长时间未命中。
- 最近多次验证失败。
- 与新规则冲突。
- 所属项目已关闭。

## 7. 冲突处理

同一主题的多条记忆可能冲突。系统必须：

- 保留来源和时间。
- 按置信度和新鲜度排序。
- 优先使用最近验证过的记忆。
- 发生重大冲突时要求人工确认或重新验证。

## 8. MemoryManager 接口

- `Recall`
- `WriteRunMemory`
- `PromoteToProjectMemory`
- `PromoteToGlobalMemory`
- `DecayOrArchive`
- `AttachMemoryContext`

### 8.1 约束

- `Recall` 必须支持作用域过滤。
- `Promote` 系列必须记录来源 run 和理由。
- `DecayOrArchive` 不应直接删除高价值记忆，应先归档。
- `WriteRunMemory` 只用于恢复和上下文压缩，不等于长期记忆写入。

## 9. 与其他子系统的关系

- `TaskClassifier`：读取少量全局记忆，用于用户偏好和默认策略。
- `Planner` 角色：读取项目记忆，避免重复试错。
- `Developer` 角色：读取仓库约定、评测基线、失败模式。
- `Reviewer` 角色：读取历史评测结果和门槛。
- `Scheduler`：可把定时任务的分类规则写为全局或项目记忆。

## 10. 边界条件

- 记忆系统不可作为唯一事实来源，事实仍以数据库、Git、评测产物和正式报告为准。
- 记忆检索失败不应阻止任务运行，只会降低效率。
- 记忆污染必须可恢复，因此所有晋升动作都必须有来源链路。
- `run` 作用域记忆默认随运行结束进入衰减或归档，不自动长期保留。
