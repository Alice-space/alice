# 08. 运维、部署与观测设计

## 1. 部署模式

### 1.1 开发模式

- 前台运行。
- 控制台可读日志。
- 本地 SQLite / BoltDB。

### 1.2 守护模式

- 单主控后台常驻。
- 结构化日志写文件。
- 周期健康检查开启。
- 调度器与 webhook 入口开启。

## 2. 配置模型

建议分层：

- 静态基线配置：项目模板、执行器默认配置、通知通道地址、集成地址。
- 敏感配置：token、密钥、SSH 凭据，走环境变量或安全存储。
- 可变运行时设置：由自然语言触发，写入数据库，例如定时任务、通知频率、观察目标、预算软阈值。
- 运行时状态：数据库中的任务、工作流、锁、游标、outbox 动作等。

边界规则：

- 自然语言可以改可变运行时设置。
- 自然语言不能直接改敏感配置和静态基线配置。
- 若用户要求变更静态基线配置，系统只生成提案，不直接落文件。

## 3. 存储布局

建议目录：

- `workspace/<project_id>/<run_id>/`
- `artifacts/<project_id>/<run_id>/`
- `state/`
- `logs/`
- `tmp/<run_id>/`

要求：

- `state/` 必须可备份。
- `logs/` 应支持按日期滚动。
- `artifacts/` 应支持按项目和运行分层查询。
- `tmp/` 可清理，但清理前要保证关键失败现场已摘要化。

## 4. 日志与审计

### 4.1 最小字段

- `timestamp`
- `level`
- `project_id`
- `task_id`
- `run_id`
- `workflow_id`
- `role`
- `executor_id`
- `event_type`
- `attempt`
- `cost`
- `duration`
- `message`

### 4.2 日志等级

- `DEBUG`
- `INFO`
- `WARN`
- `ERROR`
- `ALERT`

### 4.3 审计重点

以下动作必须进入审计：

- 审批请求与审批结果。
- 配置变更提案。
- 预算提升。
- PR 合并、删除、外部资源申请。
- 锁冲突与恢复接管。

## 5. 指标

指标必须优先选择可直接观测、可用于治理的量，而不是需要额外真值标注才能定义的理想指标。

### 5.1 系统级指标

- `fast path` 升级率。
- 人工改判率。
- 模型调用 `p50/p95` 延迟。
- 执行器成功率。
- 调度器延迟。
- 非终态运行恢复成功率。
- `Superseded` 评测比例。
- 超过 SLA 的 `Blocked` 运行数。

### 5.2 项目级指标

- 项目预算消耗与预算内完成率。
- 评测可比较率。
- 必需评测 shard 通过率。
- 任务平均迭代轮数。
- 人工介入频率。

说明：

- “任务分类命中率”这类没有稳定真值的数据，不应作为首版核心运维指标。
- 若后续引入人工标注闭环，再单独定义分类质量指标。

## 6. 健康检查

健康检查至少覆盖：

- 模型提供方连通性。
- Git MCP 连通性。
- SSH MCP 连通性。
- SLURM MCP 连通性。
- 飞书通知通道。
- 邮件适配器。
- 状态数据库读写。
- 调度器时钟漂移。
- 未完成 outbox 动作堆积。

## 7. 启动恢复与对账

恢复不能只做“数据库能打开”。主控启动时必须执行 reconciliation：

1. 扫描所有非终态 `RunRecord`。
2. 恢复或回收锁与租约。
3. 检查未完成 outbox / action intent。
4. 查询外部真实状态，例如 PR、远端作业、消息回执、游标位置。
5. 将运行标记为继续执行、补写结果、转 `Blocked`、转 `WaitingHuman` 或终止。
6. 最后再恢复调度器。

## 8. 备份与恢复

- `state/` 和记忆元数据必须定期备份。
- 产物目录可按策略分层备份。
- 恢复演练必须定期进行，不能只停留在文档里。
- 备份恢复后应优先验证锁、调度器和非终态运行的一致性。

## 9. 升级策略

首版不做自动升级，但设计需支持：

- 停机升级。
- 数据库 schema 迁移。
- 失败回滚。
- 迁移前后的恢复对账。

## 10. 运维告警

### 告警条件

- 连续模型失败。
- 执行器全不可用。
- PR 评测持续失败或长期不可比较。
- 预算耗尽。
- 主控状态库不可写。
- 未完成恢复对账超过阈值。

### 告警输出

- 飞书消息。
- 本地结构化日志。
- 后续可扩展为邮件或其他渠道。
